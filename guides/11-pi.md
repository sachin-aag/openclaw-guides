# 11 · Pi — the coding agent inside OpenClaw

OpenClaw doesn't write its own agent loop. It [embeds Pi](https://docs.openclaw.ai/pi) — an open-source coding agent maintained at [@mariozechner/pi-coding-agent](https://github.com/badlogic/pi-mono) — and wraps it in the gateway, channels, and memory subsystem.

This page is a quick orientation. The deep reference is at [docs.openclaw.ai/pi](https://docs.openclaw.ai/pi).

## Why an embedded agent (not a subprocess)

The OpenClaw integration imports Pi as a library and instantiates an `AgentSession` via `createAgentSession()`. That gives the gateway:

- Full control over session lifecycle and event handling
- Custom tool injection (messaging, sandbox, channel-specific actions)
- System-prompt customization per channel/context
- Session persistence with branching and compaction
- Multi-account auth profile rotation with failover
- Provider-agnostic model switching

Compared to spawning Pi as a CLI subprocess, the embedded approach is faster, gives finer event hooks, and lets the gateway own state.

## The four primitive tools

Pi gives every OpenClaw agent four built-in tools:

| Tool | Use | Risk level |
|---|---|---|
| `Read(path)` | Open files, scan folders, ingest context | Low |
| `Write(path, content)` | Create new files (memory, logs, reports) | Medium — can overwrite |
| `Edit(path, patch)` | Surgical patches to existing files | Medium |
| `Bash(cmd)` | Run any shell command | **High — full shell** |

OpenClaw also injects channel-specific tools (e.g., `message-tool`, `gateway-tool`, `cron-tool`) so the agent can reply, schedule jobs, or invoke other agents.

## Tool policy — the safety knob

You restrict what Pi can do via a tool policy. By default the workshop starters allow `Read`, `Write`, `Edit` against the workspace folder, and a **command allowlist** for `Bash`. To loosen or tighten:

```ts
// in your skill or gateway config
toolPolicy: {
  bash: {
    allow: ["git status", "git log --oneline -n *", "ls *", "cat notes/*"],
    deny: ["rm -rf /", "curl *", "*sudo*"],
  },
  write: {
    allowPaths: ["./workspace/**"],
  },
}
```

Production guidance: **disable `Bash` entirely** unless a skill genuinely needs it, and log every tool call.

## Models

Pi is provider-agnostic. The current OpenClaw defaults track the latest models:

- **Anthropic:** `claude-sonnet-4-6`, `claude-haiku-4-6`
- **OpenAI:** `gpt-5.4`, `gpt-5.4-mini`
- **Google:** `gemini-3.1-pro`, `gemini-3.1-flash`
- **Featherless:** any open-source model hosted on [featherless.ai](https://featherless.ai) (generous free tier; most models cap at 32K context — can be tight for agents with large memory files)

Switch by changing two lines in `.env`:

```bash
OPENCLAW_PROVIDER=openai
OPENCLAW_MODEL=gpt-5.4
```

For local models (Ollama, LM Studio), set `OPENCLAW_PROVIDER=local` and point at the OpenAI-compatible URL.

## Sessions and compaction

Each conversation produces a JSONL session file (one event per line). When the conversation gets long, Pi can compact it — replacing older turns with a summary so the context window stays affordable.

```bash
sessions/
  main:web:user-123.jsonl       # the current session
  main:web:user-123.compacted.jsonl
```

You can `tail -f` the file to watch your agent think in real time. Useful debugging move.
