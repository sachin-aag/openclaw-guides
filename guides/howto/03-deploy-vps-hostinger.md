# 03 · Deploy on Hostinger

Three doors to running OpenClaw on Hostinger, easiest first. Pick one based on how much of the stack you actually want to operate.

| Path | What you get | Cost | Time | Best for |
|------|--------------|------|------|----------|
| **A · 1-click Managed** | Hosted OpenClaw with channels, agentic mailbox, and AI credits pre-loaded. No host SSH. | ~$5.99/mo* | ~5 min | Non-technical users; "I just want it running by Monday" |
| **B · 1-click on VPS** | A KVM with the OpenClaw template + AI credits pre-installed. Full root. | ~$8.99/mo* | ~10 min | You want to customize, BYO model key, or run side projects on the box |
| **C · Manual KVM build** | Bare Ubuntu 24.04 you set up yourself: Docker, nginx, TLS, backups. Bring-your-own model key. | ~€4.99/mo | 30–60 min | Workshop participants, EU-residency-conscious users, anyone who wants to understand each layer |

<sup>* Promo pricing on the [OpenClaw landing page](https://www.hostinger.com/openclaw#pricing); both renew higher. Path C uses the standard [VPS hosting](https://www.hostinger.com/vps-hosting) plans.</sup>

> All three live on Hostinger infrastructure. They differ only in how much of the stack you operate yourself.

---

## Path A · 1-click Managed OpenClaw

Lowest-effort path. Hostinger runs the box, the OpenClaw instance, and the model billing.

> **Time:** ~5 minutes
> **Cost:** ~$5.99/mo (24-month price; renews higher — check the [pricing page](https://www.hostinger.com/openclaw#pricing))
> **What you give up:** root access, raw `docker-compose.yml`, choice of model provider, easy `restic` backups of memory files. Memory and chat logs live inside Hostinger's managed instance.

### 1 · Pick the plan

Go to [hostinger.com/openclaw](https://www.hostinger.com/openclaw#pricing) and choose **Managed OpenClaw**. AI credits are pre-loaded — you don't need an Anthropic or OpenAI account.

### 2 · Deploy

Finish the 1-click flow from your Hostinger dashboard. The instance comes up with:

- An OpenClaw web UI for managing agents and workflows
- A pre-configured agentic mailbox (so the agent has its own inbox, not yours)
- Built-in pairing for Telegram and WhatsApp
- A high-complexity security gateway generated by default

### 3 · Pair a channel

From the dashboard, pick Telegram or WhatsApp and follow the QR / bot-token flow. Send your first message — the agent should reply within seconds.

### 4 · Top up AI credits

Credits live in your Hostinger billing. Top up from the same dashboard when they run low. Watch the burn rate the first week — see [04-cron-and-heartbeat.md](../concepts/04-cron-and-heartbeat.md) for how a chatty cron job can quietly drain credits.

### Caveats (read these before you commit)

- **Memory portability.** Exporting `SOUL.md` / `MEMORY.md` happens through the UI, not `rsync`. Take periodic exports if you might leave the platform.
- **No `restic` cron.** The backup recipe in Path C does not apply.
- **Model lock-in.** You can't drop your own `ANTHROPIC_API_KEY` into a `.env` file. If you need BYO key, use Path B or C.
- **Filesystem skill access.** Many of the [workflows/](../../workflows/) starters assume access to a `workspace/` folder. Path A may or may not expose that depending on template version — confirm before you commit a workflow you've prototyped locally.

If any of those bite, jump to Path B.

## Path B · 1-click OpenClaw on VPS

Same speed at the start, but you keep the keys to the box.

> **Time:** ~10 minutes (plus optional nginx + TLS from Path C)
> **Cost:** ~$8.99/mo (renews higher — check pricing)
> **Resources:** 2 vCPU, 8 GB RAM, 100 GB NVMe, 8 TB bandwidth

### 1 · Provision

Pick **OpenClaw on VPS** at [hostinger.com/openclaw](https://www.hostinger.com/openclaw#pricing). During setup:

- **OS:** the OpenClaw template (ships on Ubuntu 24.04 LTS)
- **Datacenter:** closest to your users (Frankfurt for most EU readers)
- **SSH key:** upload your public key. **Don't use password auth.**

### 2 · SSH in

```bash
ssh root@<your-ip>
```

The OpenClaw template is already running. Find it:

```bash
docker ps
```

The template's compose file and `.env` live in a project directory (commonly `/root/openclaw` or `/opt/openclaw` — exact path varies by template version). `cd` into the directory the running container reports and you'll see the familiar `docker-compose.yml` and `.env`.

### 3 · Swap in your own model key (optional)

By default the template uses Hostinger's pre-loaded AI credits. To bring your own model key:

```bash
cd <template-dir>
nano .env
```

Set:

```bash
OPENCLAW_PROVIDER=anthropic
OPENCLAW_MODEL=claude-sonnet-4-6
ANTHROPIC_API_KEY=sk-ant-...
```

Then:

```bash
docker compose up -d
```

### 4 · Harden + TLS (recommended)

The template gives you a working OpenClaw, not a hardened public-facing server. Walk steps **2 (harden)**, **5 (nginx + TLS)**, and **6 (backups)** of Path C below — they all apply unchanged.

## Path C · Manual KVM build

The DIY path. This is what the workshop teaches and what gives you the cheapest €/year. Where you put OpenClaw when you want it always-on, cron actually fires, and you control the box. Hostinger's KVM plans are the cheapest sane option in the EU.

> **Time:** ~30–60 minutes (first time)
> **Cost:** KVM 1 ≈ €4.99/mo, KVM 2 ≈ €6.99/mo
> **Best for:** personal always-on agents, small team deployments, anyone wanting an EU-hosted box.

### Prerequisites

- A Hostinger account ([hostinger.com/vps-hosting](https://www.hostinger.com/vps-hosting))
- An SSH key pair on your laptop (`ssh-keygen -t ed25519` if you don't have one)
- Your model API key
- A domain name (optional but recommended for TLS)

### 1 · Provision the VPS

In the Hostinger dashboard:

1. **Plan** → KVM 1 (1 vCPU, 4 GB RAM, 50 GB SSD) is enough for a personal agent. KVM 2 if you'll run multiple agents or use a heavier model.
2. **OS** → **Ubuntu 24.04 LTS**.
3. **Datacenter** → pick the one closest to your users (Frankfurt for most EU readers).
4. **SSH key** → upload your public key during setup. **Don't use password auth.**
5. Wait ~3 minutes for provisioning. You'll get an IP address.

### 2 · First login + harden

```bash
ssh root@<your-ip>
```

Run the basics:

```bash
# create a non-root user
adduser openclaw
usermod -aG sudo openclaw

# copy your SSH key to that user
rsync --archive --chown=openclaw:openclaw ~/.ssh /home/openclaw

# disable root SSH + password auth
sed -i 's/^#\?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^#\?PasswordAuthentication.*/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart ssh

# basic firewall
apt update && apt install -y ufw fail2ban
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
systemctl enable --now fail2ban
```

Log out, log back in as `openclaw`:

```bash
ssh openclaw@<your-ip>
```

### 3 · Install Docker + Docker Compose

```bash
sudo apt install -y docker.io docker-compose-plugin
sudo usermod -aG docker openclaw
# log out + back in to pick up the group
exit
ssh openclaw@<your-ip>
docker --version
```

### 4 · Pull the OpenClaw image

```bash
mkdir -p ~/openclaw && cd ~/openclaw
```

Create `docker-compose.yml`:

```yaml
services:
  openclaw:
    image: ghcr.io/openclaw/openclaw:latest   # or pin a version
    container_name: openclaw
    restart: unless-stopped
    ports:
      - "127.0.0.1:3000:3000"   # bind to localhost; nginx fronts it
    env_file: .env
    volumes:
      - ./workspace:/app/workspace
      - ./sessions:/app/sessions
```

Create `.env` (same keys as the local install — see [01-install-local.md › step 3](01-install-local.md)):

```bash
OPENCLAW_PROVIDER=anthropic
OPENCLAW_MODEL=claude-sonnet-4-6
ANTHROPIC_API_KEY=sk-ant-...
WORKSPACE_DIR=/app/workspace
OPENCLAW_HOST=0.0.0.0
```

Start it:

```bash
docker compose up -d
docker compose logs -f
```

You should see the gateway boot. Hit `Ctrl-C` to stop tailing logs (the container keeps running).

### 5 · Front it with nginx + TLS

Skip if you only want to use SSH tunneling. Recommended if you want a real URL.

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

Point a DNS A record (`agent.yourdomain.com`) at the VPS IP. Then:

```bash
sudo nano /etc/nginx/sites-available/openclaw
```

```nginx
server {
    listen 80;
    server_name agent.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/openclaw /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d agent.yourdomain.com
```

You now have HTTPS at `https://agent.yourdomain.com`.

> ⚠ **Do not** put OpenClaw on the public internet without auth in front of it. Either restrict by IP, or add basic auth in the nginx block, or use a reverse proxy with OAuth (Authelia, Pomerium, Cloudflare Zero Trust).

### 6 · Backups

Memory files are the only thing that's hard to recreate. Back them up nightly:

```bash
sudo apt install -y restic
restic init --repo /var/backups/openclaw   # local
# ...or to B2 / S3 / SFTP — see restic docs
```

```bash
# /etc/cron.daily/openclaw-backup
#!/bin/bash
restic --repo /var/backups/openclaw backup /home/openclaw/openclaw/workspace
restic --repo /var/backups/openclaw forget --keep-daily 7 --keep-weekly 4 --prune
```

```bash
sudo chmod +x /etc/cron.daily/openclaw-backup
```

### Updating OpenClaw

```bash
cd ~/openclaw
docker compose pull
docker compose up -d
```

### Cost ceiling

KVM 1 plus an EU TLD is ~€60/year. Model API on top of that depends on usage; budget €5–20/mo for personal use. Set a hard cap in your provider dashboard.

## Which path should you actually pick?

- **You opened this guide because someone said "just deploy it" and you don't want a server.** → Path A.
- **You want it deployed today and customized tomorrow.** → Path B.
- **You're here for the workshop, or you want the full mental model of how OpenClaw runs in production.** → Path C.

You can also start at A and migrate to C later — your `SOUL.md` and `MEMORY.md` are portable, just export from the UI before you go.
