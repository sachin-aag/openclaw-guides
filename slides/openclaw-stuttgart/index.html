05 / 15Architecture · openclaw.cc/concepts
Core concept · 1/4

Gateway.

The central hub. One normalized language in, one out.

  • Routes messages from any channel to the right agent and session.
  • Holds the agent loop: receive → run → reply → persist.
  • Owns the heartbeat: cron jobs and scheduled wake-ups live here.
  • You don't write a gateway. You configure one and forget about it.

07 / 15Channels
Core concept · 3/4

Memory is markdown on disk.

Your agent's personality, preferences, and long-term knowledge live in plain markdown files you open in any editor.

No vector DB to set up. No black box. cat, vim, git diff.

SOUL.md is an official OpenClaw template.

# SOUL.md  ·  who the agent is

You are Lobster, a calm second-brain
for Sachin. Speak briefly. Save
anything tagged #keep to MEMORY.md.

# MEMORY.md  ·  what it has learned

- 2026-05-08  Sachin prefers
  short morning briefings, no emojis.
- 2026-05-09  OpenClaw event
  in Stuttgart confirmed for tonight.
08 / 15Memory · SOUL.md
Core concept · 4/4

Always on. Not waiting.

Most "AI assistants" only think when you ping them. OpenClaw can run on a heartbeat.

  • Wake every minute, hour, or day.
  • Read its memory, check the world, write back.
  • Send you a message — not the other way around.
# cron · the daily review agent

every day at 07:30:
  read notes/*.md from yesterday
  summarize into daily-review.md
  message me on the web channel

# this is the canonical "always-on" pattern
09 / 15Cron · Heartbeat
Inside the agent

Pi — the coding brain.

OpenClaw embeds Pi, an open-source coding agent that gives your assistant real hands. Read, Write, Edit, Bash — the same primitives a developer uses.

read

Read

Open files, scan folders, ingest context.

write

Write

Create new files. Memory. Logs. Reports.

edit

Edit

Surgical patches. Update one line. Bump a counter.

bash

Bash

Run anything. Shell, scripts, your own tools.

Why it's shaped this way

Four principles.

01

Local-first

Runs on your laptop, your VPS, or your homelab. The cloud is optional.

02

Privacy-focused

Your memory, your messages, your model keys. Nothing routed through someone else's gateway.

03

Extensible

New tools, new skills, new channels — all pluggable. Nothing in the core is sacred.

04

Multi-platform

One agent, every surface where your humans actually are.

Wider landscape

We focus on OpenClaw. Hermes is worth a look too.

Hermes (Nous Research, Feb 2026) is the closest peer to OpenClaw — and on a few axes it's been pulling ahead. Know both, pick one.

OpenClaw

Open-source agent gateway. Markdown memory. Embeds the Pi coding agent. TypeScript. MIT.

Best when: editable memory, minimal moving parts, TS shop.

Hermes

Open-source autonomous agent. Persistent memory. Automated skill creation. Multi-channel gateway (Telegram, Discord, Slack, WhatsApp, Signal, CLI). 40+ built-in skills. Local / Docker / SSH / Modal sandboxes. Python. MIT.

Best when: richer skills catalog, stronger sandboxing, Python shop.

hermes-agent.org · github.com/badlogic/pi-mono · openclaw.cc
